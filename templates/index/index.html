Local web application template for desktop apps

This project is a ready-to-use foundation for building local desktop applications with a web-based user interface. The application runs a Flask backend on the local machine and displays the interface inside a native WebView window, giving you the development comfort of a web app while keeping the distribution model close to a desktop application.

The template separates the application into focused layers: controllers expose HTTP endpoints, data models define typed structures and JSON serialization, storage classes persist application data, and the WebView runtime chooses the best available backend for macOS, Windows, or Linux. The main window remembers its last size and position by saving settings to a platform-specific configuration directory.

Built-in storage uses thread-safe JSON access with atomic writes, cache invalidation, deep-copied snapshots, and default data initialization when the settings file is missing or invalid. The settings API is available through simple JSON endpoints, so frontend code can load and save runtime configuration without knowing where or how the file is stored on disk.

The build script is prepared for system-specific packaging. It can create native-style application outputs for macOS, Windows, and Linux, attach metadata such as version, author, copyright, description, and application icon, and package the generated artifacts into release archives suitable for distribution.

To use the template, customize the values in the configuration file, add your own controllers, models, storage classes, templates, and static assets, then run the application locally during development. When the app is ready, run the build script on the target operating system to generate the desktop package for that platform.
